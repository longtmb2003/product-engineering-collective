# Visual Language

> This document defines how the website should look and feel.
>
> It is not a UI specification.
> It defines the visual principles behind every design decision.
>
> Every component should follow this language.

---

# Design Principles

The interface should feel

- Calm
- Premium
- Minimal
- Intentional
- Engineering-driven

The design should communicate confidence through restraint rather than decoration.

Everything should feel carefully designed.

Nothing should feel excessive.

---

# Design References

The following products are references for quality and craftsmanship.

They are references for thinking, not templates.

- Linear
- Vercel
- Railway
- Stripe
- Raycast
- Supabase

Do not copy layouts.

Do not imitate interactions.

Understand why their interfaces feel premium.

---

# Layout Philosophy

Every section should breathe.

Prefer whitespace over decoration.

Large spacing creates confidence.

Avoid visual clutter.

Content should never feel compressed.

Every page should naturally guide the eye downward.

---

# Grid System

Desktop

- Maximum width: 1280px
- Comfortable reading width for text
- Consistent gutters

Tablet

- Fluid layout

Mobile

- Mobile-first
- Single-column by default

---

# Spacing Rhythm

Spacing should follow a consistent scale.

Use spacing to establish hierarchy.

Avoid arbitrary values.

Sections should feel clearly separated.

Cards should have generous internal padding.

Whitespace is considered a design element.

---

# Typography

Typography carries most of the visual identity.

Use large headings.

Avoid long paragraphs.

Prefer short readable blocks.

Headings should feel confident.

Body text should remain highly readable.

Hierarchy should be immediately obvious.

---

# Color Philosophy

Dark-first.

Use color with intention.

Accent colors should highlight actions, not decorate the interface.

Large colorful gradients should never overpower content.

Contrast must remain accessible.

---

# Surfaces

Prefer layered surfaces.

Background

↓

Section

↓

Card

↓

Interactive Element

Each layer should be visually distinguishable while remaining subtle.

---

# Borders

Borders should be soft.

Avoid heavy outlines.

Borders exist to define structure rather than attract attention.

---

# Corners

Use consistent border radius across the entire website.

Cards

Buttons

Inputs

Images

All should feel part of one system.

---

# Shadows

Shadows should create depth, not drama.

Avoid large floating shadows.

Depth should remain subtle.

---

# Backgrounds

Backgrounds should remain quiet.

Possible elements

- soft gradients
- grid textures
- subtle noise
- radial light
- blurred glow

Backgrounds must never compete with content.

---

# Motion Philosophy

Motion should explain.

Not entertain.

Animation exists to

- guide attention
- reinforce hierarchy
- communicate relationships

Avoid animation for decoration alone.

---

# Motion Principles

Prefer

- fade
- slide
- scale
- opacity
- blur

Avoid

- bounce
- elastic
- spinning
- excessive parallax

Movement should feel smooth and deliberate.

---

# Images

Images should support credibility.

Preferred

- Product screenshots
- Device mockups
- Team portraits
- Architecture diagrams

Avoid

- Generic stock photos
- Fake office photography
- AI-generated business imagery

Every image should represent real work.

---

# Product Presentation

Products are the strongest proof of engineering capability.

Each featured product should include

- Cover image
- Brief story
- Engineering highlights
- Technologies
- Links

Focus on solving problems rather than listing features.

---

# Team Photography

Portraits should feel authentic.

Consistent framing.

Consistent lighting.

Simple background.

Avoid excessive editing.

The goal is professionalism rather than perfection.

---

# Iconography

Icons should remain simple.

Consistent stroke weight.

Support content rather than dominate it.

---

# Cards

Cards should feel spacious.

Content should have room to breathe.

Avoid dense layouts.

Hover effects should be subtle.

Cards should never become miniature landing pages.

---

# Interaction Feedback

Buttons should provide immediate feedback.

Links should clearly indicate interactivity.

Hover effects should remain subtle.

Transitions should feel fast but calm.

---

# Responsive Design

The mobile experience is not a reduced desktop.

Layouts should adapt naturally.

Prioritize readability over density.

Touch targets must remain comfortable.

---

# Accessibility

Contrast should always remain readable.

Typography should never rely solely on color.

Motion should respect reduced-motion preferences.

Keyboard navigation should be fully supported.

Accessibility is part of quality.

---

# Overall Feeling

When someone visits the website, they should think

"This feels like a team that cares about engineering."

Not

"This website has impressive animations."

Engineering quality should always remain the hero.
